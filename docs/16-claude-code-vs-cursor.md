# Cursor vs Claude Code: parity matrix and gaps

The toolkit ships for both Cursor and Claude Code from a single canonical source (`.cursor/`). The `.claude/` tree is regenerated by `scripts/build-claude-target.py` (idempotent; CI enforces). This doc records the translation matrix, the chosen patterns, and the known gaps.

## Authoritative spec references

- Cursor: [Rules](https://docs.cursor.com/en/context/rules) · [Agents](https://docs.cursor.com/en/agents/overview) · [Hooks](https://docs.cursor.com/en/agents/hooks) · [Slash commands](https://docs.cursor.com/en/agents/slash-commands)
- Claude Code: [Subagents](https://docs.claude.com/en/docs/claude-code/sub-agents) · [Skills](https://docs.claude.com/en/docs/claude-code/skills) · [Slash commands](https://docs.claude.com/en/docs/claude-code/slash-commands) · [Hooks](https://docs.claude.com/en/docs/claude-code/hooks) · [Settings](https://docs.claude.com/en/docs/claude-code/settings) · [MCP](https://docs.claude.com/en/docs/claude-code/mcp)

## Translation matrix

| Concept | Cursor | Claude Code | Notes |
|---|---|---|---|
| Auto-activating rules | `.cursor/rules/*.mdc` (`alwaysApply`, `globs`) | `.claude/skills/rule-<name>/SKILL.md` | All 39 rules in this repo use narrow `globs` (none are `alwaysApply: true`), so every rule becomes a skill. Description gets a `MANDATORY when editing files matching <globs>` prefix so Claude will surface it. |
| Workflow agents & specialists | `.cursor/agents/*.md` | `.claude/agents/*.md` | Frontmatter rewritten: drop `readonly`/`is_background`; map `model: inherit` → `sonnet` (orchestrators in `OPUS_AGENTS` get `opus`). |
| Skills | `.cursor/skills/<name>/SKILL.md` (+ helpers) | `.claude/skills/<name>/SKILL.md` (+ helpers) | Helpers copied verbatim. Skill frontmatter normalized to `name` + `description`. |
| Slash commands | `.cursor/commands/*.md` | `.claude/commands/*.md` | Bodies copied as-is. No Cursor-specific argument tokens to translate in this repo. |
| Hooks | `.cursor/hooks.json` + `.cursor/hooks/` | `.claude/settings.json` `hooks` block + `.claude/hooks/` | See event mapping below. Cursor hook scripts (Bun/TS) are reused unchanged from `.claude/hooks/`. |
| Notepads | `.cursor/notepads/*.md` | `.claude/skills/notepad-<name>/SKILL.md` | Wrapped as reference-doc skills. |
| Snippets | `.cursor/snippets/*.code-snippets` | _(not ported)_ | VS Code snippets work in any VS Code-based editor regardless of assistant; no Claude Code analog. |
| MCP servers | `.cursor/mcp.json` | `.mcp.json` (repo root) | Same `mcpServers` schema. Generator copies the file verbatim. |
| Project memory | `.cursorrules` | `.claude/CLAUDE.md` | Generated from `.cursorrules` with `.cursor/` paths rewritten + an index of rule-skills appended. |

## Hook event mapping

| Cursor event | Claude Code event | Matcher | Status |
|---|---|---|---|
| `beforeShellExecution` | `PreToolUse` | `Bash` | ✅ Mapped |
| `afterFileEdit` | `PostToolUse` | `Edit\|Write\|MultiEdit` | ✅ Mapped |
| `stop` | `Stop` | — | ✅ Mapped |
| `beforeMCPExecution` | _(none)_ | — | ⚠️ Gap — no first-class analog. The script is left in `.claude/hooks/` for manual invocation. |
| `beforeReadFile` | _(none)_ | — | ⚠️ Gap — Claude Code does not expose a pre-read hook. Secret-redaction must be handled at the file source. |
| `beforeSubmitPrompt` | `UserPromptSubmit` (closest) | — | ⚠️ Not auto-mapped. Verify the script is safe to run on every prompt before enabling. |

## Source-of-truth pattern

- `.cursor/` is **the canonical source**. Authors edit there.
- `scripts/build-claude-target.py` regenerates `.claude/` and `.mcp.json` from `.cursor/`.
- The script runs automatically inside `setup.sh` when target is `claude` or `both`.
- CI (`.github/workflows/claude-target-drift.yml`) runs `--check` on every push that touches `.cursor/`, `.claude/`, `.mcp.json`, or the script itself, and fails if the trees drift.
- The generated `.claude/` tree is committed so users without Python can still install.

## Known gaps and design choices

1. **Three Cursor hook events have no Claude Code analog** (see mapping table). Their scripts ship in `.claude/hooks/` but are not auto-registered. Document on `.claude/hooks/README.md`.
2. **Snippets are not ported.** They work in any VS Code-based editor (including Cursor) and need no equivalent in Claude Code, which has no snippet primitive.
3. **All rules are realized as skills, not as `CLAUDE.md` sections.** Per-rule narrow globs map cleanly to skill MANDATORY-TRIGGER descriptions, and inlining 39 rules into `CLAUDE.md` would load them on every turn.
4. **Hooks still require Bun.** No change vs Cursor.
