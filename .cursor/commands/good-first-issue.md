# Good First Issue

Identify small, well-scoped contribution opportunities in simpler-grants-gov and draft GitHub issue descriptions with clear acceptance criteria and a named learning outcome.

## What I Need From You

1. **Area** — `api`, `frontend`, `infra`, `docs`, or a specific path
2. **Theme** (optional) — error handling, testing, docstrings, i18n, accessibility, lint cleanup
3. **Count** (optional) — number of issues to draft (default 1)

## What Happens Next

The Good First Issue Agent will:
1. Load architecture + convention context for the target area
2. Grep candidate files matching the theme
3. Filter candidates down to real 1–2 hour tasks
4. Draft each issue with context, acceptance criteria, learning outcome, and links to the governing rule file and an example PR
5. Run convention, scope, and duplication quality gates
6. Present the markdown for your review

The agent does not open issues on GitHub — you copy the markdown when you're ready.

## Tips for Better Results
- Narrow the area to a single domain
- Name a theme — untargeted scans produce noisy candidates
- Pair with `/onboarding` if you are seeding the backlog for a cohort of new contributors
